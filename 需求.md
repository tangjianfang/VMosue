## 2. 关键现实约束

### 2.1 “99% 以上”需要重新定义

如果目标是“在任何环境、任何摄像头、任何用户、任何光照下，所有手势 99% 准确”，这个不现实。

更可行的定义是：

| 指标 | MVP 目标 |
|---|---:|
| 光标移动可用率 | ≥ 99% |
| 左键单击成功率 | ≥ 99% |
| 右键单击成功率 | ≥ 97% |
| 双击成功率 | ≥ 95% |
| 拖拽成功率 | ≥ 95% |
| 误触发率 | ≤ 1% |
| 端到端延迟 | P95 < 60ms |
| 丢手恢复时间 | < 300ms |

这里的前提是：

- 光照正常。
- 摄像头能稳定看到双手。
- 用户完成 30-60 秒校准。
- 用户使用推荐手势。
- 摄像头至少 720p 30fps，最好 720p 60fps。

### 2.2 纯“空中点击”不是最高可靠方案

你提到“右手手指在空中点击一下，相当于鼠标右键点击一次”。

这个可以做，但需要注意：普通单目摄像头没有真实深度，只能通过 MediaPipe 的相对 Z 值、手指长度变化、指尖速度等特征估计“向前点了一下”。这种手势体验自然，但稳定性通常不如“拇指食指捏合”。

所以建议：

- 最高可靠操作，例如左键点击、拖拽，用“捏合”完成。
- 空中点击作为右键或快捷操作。
- 如果空中点击识别置信度不足，自动降级为二指点击或捏合右键。
- App 内提供“空中点击灵敏度校准”。

这样才能接近你要的“极佳体验”。

## 3. 推荐交互设计

### 3.1 左右手分工

推荐采用：

- 右手：主鼠标手，负责光标、点击、拖拽、右键、双击。
- 左手：辅助控制手，负责滚动、暂停、模式切换、修饰键。

这样做的好处：

- 避免所有动作都压在右手上，降低误触发。
- 让点击和滚动解耦。
- 接近真实鼠标 + 键盘的使用逻辑。

## 4. 右手操作设计

### 4.1 光标移动

| 项目 | 设计 |
|---|---|
| 手势 | 右手食指伸出，其他手指自然收起 |
| 控制点 | 不建议直接用食指尖，建议用“食指根部 + 掌心方向”综合估计 |
| 原因 | 食指尖点击时会抖，直接拿来控制光标会导致点击偏移 |
| 映射 | 摄像头控制区映射到整个屏幕 |
| 平滑 | One Euro Filter / 卡尔曼滤波 |
| 边缘处理 | 中央控制区放大到全屏，边缘预留缓冲区 |

光标控制不能简单做：

```text
食指尖坐标 = 鼠标坐标
```

更好的方式是：

```text
手掌稳定点控制光标位置
食指尖/手指动作负责点击判断
```

这样点击时光标不会被手指快速动作带偏。

---

### 4.2 左键单击

为了达到 99% 体验，左键单击建议用：

| 项目 | 设计 |
|---|---|
| 手势 | 右手拇指 + 食指快速捏合一次 |
| 动作 | 鼠标左键单击 |
| 判定 | 拇指尖与食指尖距离快速缩小后释放 |
| 优点 | 单目摄像头下最稳定 |
| 反馈 | 点击瞬间显示圆环 + 声音提示 |

为什么推荐捏合？

- 指尖距离是 2D 几何关系，识别稳定。
- 不依赖真实深度。
- 不容易和光标移动混淆。
- 用户学习成本低。

---

### 4.3 右键单击

你提到的“右手手指空中点击一下 = 鼠标右键点击一次”，可以作为右键动作。

| 项目 | 设计 |
|---|---|
| 手势 | 右手食指向前轻点一次 |
| 动作 | 鼠标右键单击 |
| 检测方式 | Z 轴变化 + 指尖速度 + 手指长度变化 + 状态机 |
| 判定窗口 | 120-250ms |
| 冷却时间 | 250-350ms |
| 反馈 | 右键菜单前显示蓝色点击波纹 |

但需要加一个可靠性保护：

- 如果系统识别不到明确的“前后点击模式”，不触发。
- 不用“疑似点击”触发右键。
- 允许用户在设置中改成“中指 + 食指捏合 = 右键”。

推荐默认方案：

| 操作 | 默认手势 | 备选手势 |
|---|---|---|
| 左键 | 拇指食指捏合 | 空中轻点 |
| 右键 | 食指空中点击 | 二指捏合 |
| 双击 | 左键手势连续两次 | 空中点击连续两次 |

这样既满足你的空中点击需求，又保证基本操作可靠。

---

### 4.4 双击

双击不要直接依赖系统双击判定，建议 App 内自己做时序判断。

| 项目 | 设计 |
|---|---|
| 手势 | 连续两次左键手势 / 连续两次空中点击 |
| 时间窗口 | 250-500ms 可配置 |
| 空间约束 | 两次点击坐标距离小于一定阈值 |
| 输出 | 直接发送双击事件 |
| 防误触 | 第一次点击后进入等待窗口 |

双击有两种策略：

方案 A：立即单击，然后第二次来时补成双击  
体验快，但可能先触发单击副作用。

方案 B：第一次点击后等待 250ms，确认是否双击  
更安全，但单击有延迟。

我建议：

- 默认使用方案 A，保证鼠标体验跟手。
- 对某些场景，例如文件管理器，可以提供“安全双击模式”。

---

### 4.5 拖拽

| 项目 | 设计 |
|---|---|
| 手势 | 捏合并保持 |
| 动作 | 鼠标左键按下并拖动 |
| 开始条件 | 捏合持续 150-250ms |
| 结束条件 | 手指分开 |
| 安全处理 | 丢手时自动释放鼠标按键 |
| 反馈 | 光标环变为实心/变色 |

拖拽不能用握拳作为第一选择，因为握拳会让关键点少、手形变化大，容易导致光标丢失。

推荐：

- 捏合短按 = 左键单击。
- 捏合保持 = 拖拽。
- 松开捏合 = 释放鼠标。

这是最接近触控板的交互。

## 5. 左手操作设计

### 5.1 滚动

| 项目 | 设计 |
|---|---|
| 手势 | 左手食指/双指伸出 |
| 动作 | 上下移动控制滚轮 |
| 映射 | 左手向上 = 页面向上 / 可配置反向 |
| 模式 | 左手进入滚动姿态后，右手仍可控制光标 |
| 反馈 | 屏幕侧边显示滚动条动画 |

滚动最好不要让右手完成，因为右手已经负责定位和点击。

推荐：

```text
左手双指伸出 → 进入滚动模式
左手上下移动 → 滚轮
左手放下/张开 → 退出滚动模式
```

---

### 5.2 暂停 / 恢复

一定要有暂停机制，否则误触发会很烦。

| 手势 | 动作 |
|---|---|
| 左手张开保持 1 秒 | 暂停手势控制 |
| 左手再次张开保持 1 秒 | 恢复手势控制 |
| 双手张开保持 1 秒 | 紧急暂停 |
| 物理键盘快捷键 | 立即退出/暂停 |

建议加一个全局热键：

```text
Ctrl + Alt + G = 启用/暂停手势控制
Esc 长按 = 紧急停止
```

---

### 5.3 辅助键

MVP 可以保留少量辅助键，但不要做太多。

| 左手手势 | 动作 |
|---|---|
| 左手握拳保持 | Shift 按住 |
| 左手拇指食指捏合 | Ctrl 按住 |
| 左手张开 | 释放所有辅助键 |

这可以支持：

- Ctrl + 点击
- Shift + 点击
- 多选
- 拖选

但第一版不要加复制、粘贴、复杂快捷键，否则手势冲突会明显增加。

## 6. 推荐最终 MVP 手势表

### 6.1 默认手势

| 操作 | 默认手势 | 优先级 |
|---|---|---:|
| 移动光标 | 右手食指指向并移动 | 最高 |
| 左键单击 | 右手拇指 + 食指快速捏合 | 最高 |
| 左键双击 | 右手捏合连续两次 | 高 |
| 拖拽 | 右手捏合保持并移动 | 高 |
| 右键单击 | 右手食指空中点击一次 | 高 |
| 右键双击 | 右手食指空中点击两次 | 中 |
| 滚动 | 左手双指伸出并上下移动 | 高 |
| 暂停/恢复 | 左手张开保持 1 秒 | 最高 |
| 紧急停止 | 双手张开保持 1 秒 / 热键 | 最高 |

### 6.2 为什么这样设计

- 左键、拖拽这些最常用动作使用最高可靠的捏合。
- 右键使用你希望的“空中点击”。
- 双击由 App 做时序识别，而不是完全交给系统。
- 左手只做辅助，降低认知负担。
- 暂停手势始终最高优先级，保证安全感。

## 8. 手势识别核心方案

### 8.1 不做“单帧识别”，必须做状态机

单帧识别会抖、会误触发。正确方式是：

```text
检测到手势趋势
  ↓
进入候选状态
  ↓
连续 N 帧满足条件
  ↓
触发动作
  ↓
进入冷却状态
  ↓
等待释放
```

比如点击状态机：

```text
IDLE
  ↓ 手指开始捏合
PRESSING
  ↓ 捏合距离小于阈值
PRESSED
  ↓ 手指释放
CLICK_CONFIRMED
  ↓ 冷却结束
IDLE
```

这样可以极大减少误触发。

---

### 8.2 空中点击检测

空中点击需要融合多个特征，不能只看一个 Z 值。

推荐特征：

| 特征 | 用途 |
|---|---|
| 食指尖 Z 相对变化 | 判断向前/向后 |
| 食指尖 Y 速度 | 判断点击动作方向 |
| 食指长度投影变化 | 判断手指是否前伸 |
| 指尖到掌心距离变化 | 辅助判断 |
| 手掌整体位移 | 排除整只手移动造成的误判 |
| 时间窗口 | 限定点击动作时长 |

判定逻辑应该是：

```text
手掌整体稳定
  + 食指尖快速前伸
  + 食指尖快速回弹
  + 持续时间合理
  + 置信度超过阈值
  = 空中点击
```

必须排除这种情况：

```text
用户移动手控制光标
```

否则光标移动时会误判成点击。

---

### 8.3 光标平滑

光标平滑不能用简单大窗口平均，因为会增加延迟。

推荐：

| 算法 | 用途 |
|---|---|
| One Euro Filter | 首选，低速稳定，高速跟手 |
| 卡尔曼滤波 | 可用于预测 |
| Dead Zone | 去除微抖 |
| 动态灵敏度 | 快速移动时加速，精确操作时减速 |

理想体验应该是：

- 快速移动时：光标跟得上。
- 停下来时：光标不抖。
- 小范围调整时：精细。
- 点击时：光标不漂移。

## 9. Windows 输入注入方案

### 9.1 MVP 使用 SendInput

Windows 上最稳妥的 MVP 方案是 `SendInput`。

优点：

- 系统原生。
- 延迟低。
- 不需要驱动。
- 开源项目可直接使用。
- 安装简单。
- 对普通桌面应用兼容性好。

限制：

- 无法控制 UAC 安全桌面。
- 普通权限 App 可能无法控制管理员权限窗口。
- 某些游戏/反作弊可能拦截。
- 不是真正的内核级虚拟 HID 设备。

### 9.2 为什么 MVP 不建议做虚拟 HID 驱动

真正的 Windows 虚拟 HID 鼠标/键盘需要驱动，问题很多：

- 需要驱动签名。
- 安装复杂。
- 杀毒软件可能拦截。
- 用户信任成本高。
- 开源免费维护成本大。
- Windows 版本兼容成本高。

所以建议：

- MVP：`SendInput`。
- 后续专业版：可选虚拟 HID 驱动。
- 对游戏/特殊应用：单独评估。

## 10. App 体验设计

### 10.1 App 形态

推荐做成：

- 托盘常驻 App。
- 启动后显示一个小型控制面板。
- 可打开实时调试窗口。
- 平时只显示轻量 Overlay。
- 支持开机启动。
- 支持一键暂停/恢复。

不要用 OpenCV 的 `imshow` 做正式界面，那只是调试工具。

---

### 10.2 Overlay 反馈

为了让用户觉得“可靠”，必须有反馈。

建议显示：

| 反馈 | 作用 |
|---|---|
| 虚拟光标环 | 显示系统当前识别的控制点 |
| 点击波纹 | 告诉用户点击已触发 |
| 手势状态角标 | MOVE / CLICK / DRAG / SCROLL |
| 识别置信度颜色 | 绿色稳定，黄色临界，红色丢失 |
| 暂停状态提示 | 防止用户不知道当前是否生效 |
| 滚动方向提示 | 提升滚动可控感 |

好的反馈可以显著提升“主观准确率”。

---

### 10.3 首次校准

如果要做到接近 99%，首次校准必须有。

校准流程建议 30-60 秒：

1. 选择主控手：默认右手。
2. 识别摄像头位置。
3. 让用户把手移动到屏幕四角，建立控制区映射。
4. 让用户做 5 次捏合，学习点击阈值。
5. 让用户做 5 次空中点击，学习 Z 轴阈值。
6. 让用户做一次拖拽。
7. 让用户做一次滚动。
8. 保存个人配置。

这样可以适配：

- 手大小。
- 摄像头角度。
- 用户习惯。
- 手势幅度。
- 光照环境。

## 13. 风险与取舍

### 13.1 最大风险：空中点击可靠性

风险：

- 单目摄像头深度不准。
- 不同用户动作幅度差异大。
- 光标移动可能误判为空中点击。

缓解：

- 空中点击只做右键或可选动作。
- 左键用捏合保证基础体验。
- 强制做空中点击校准。
- 加状态机和冷却时间。
- 提供备选右键手势。

---

### 13.2 最大体验风险：手部疲劳

长期悬空手势会累。

缓解：

- 控制区不要太大。
- 支持手肘放桌面。
- 使用小幅度动作。
- 支持灵敏度调节。
- 支持暂停模式。
- 左右手分工，减少单手负担。

---

### 13.3 最大系统风险：输入权限

Windows 下：

- 普通 App 不一定能控制管理员权限窗口。
- UAC 安全桌面不能控制。
- 某些游戏不能控制。

缓解：

- 明确提示用户。
- 提供“以管理员身份运行”选项。
- 不在 MVP 里承诺游戏/安全桌面。

## 16. 我建议的最终 MVP 决策

基于你的要求，我建议定稿如下：

```text
产品形态：
Windows 原生桌面 App，托盘常驻，轻量 Overlay。

技术路线：
C++ + OpenCV + MediaPipe/ONNX + Win32 SendInput。

手势策略：
右手主控光标和点击，左手控制滚动和暂停。

核心交互：
右手食指移动光标；
右手拇指食指捏合 = 左键；
右手捏合保持 = 拖拽；
右手食指空中点击 = 右键；
右手连续点击/捏合 = 双击；
左手双指上下移动 = 滚轮；
左手张开保持 = 暂停/恢复。

体验策略：
必须有校准、状态机、视觉反馈、音效反馈、误触保护、紧急停止。

性能策略：
原生 App、多线程、低分辨率推理、最新帧优先、One Euro Filter、空闲降频。

开源策略：
Apache-2.0，避免 GPL 依赖。
```

这个边界是可落地的，而且最符合“性能优先 + 体验优先 + Windows App + 开源免费”。
